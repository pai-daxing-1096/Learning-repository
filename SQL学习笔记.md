### 三.DDL语句

#### 1.数据库操作

##### (1)查询

查询当前数据库信息

###### ①查询所有数据库

语法：

`SHOW DATABASES;`

###### ②查询当前数据库

语法：

`SELECT DATABASE();`

##### (2)创建

创建数据库

语法：

`CREATE DATABASE [IF NOT EXISTS] 数据库名 [DEFAULT CHARACTER SET 字符集] [COLLATE 排序规则];`

- `IF NOT EXISTS`   如果没有该数据库则创建

- 字符集：

	| 字符集名称    | 描述                 | 最大字节/字符 | 默认排序规则 (MySQL 8.0) | 适用场景                                          |
	| :------------ | :------------------- | :------------ | :----------------------- | :------------------------------------------------ |
	| **`utf8mb4`** | UTF-8 Unicode (推荐) | 4             | `utf8mb4_0900_ai_ci`     | **首选**。支持所有语言、特殊符号和Emoji。         |
	| `utf8`        | UTF-8 Unicode        | 3             | `utf8_general_ci`        | **不推荐**。它是`utf8mb3`的别名，无法存储Emoji 。 |
	| `latin1`      | cp1252 西欧语言      | 1             | `latin1_swedish_ci`      | 仅用于处理非常老的系统或纯英文/西欧数据。         |
	| `ascii`       | 美国ASCII            | 1             | `ascii_general_ci`       | 只包含基本英文字母、数字和符号。                  |
	| `gbk`         | GBK 简体中文         | 2             | `gbk_chinese_ci`         | 仅用于处理简体中文数据，且无需Emoji的场景。       |
	| `gb18030`     | 国家标准GB18030      | 4             | `gb18030_chinese_ci`     | 用于需要符合中国国家标准的中文系统。              |

- 排序规则：

	| `_ai`  | 不区分重音   | `a` 和 `á` 被视为相同                            |
	| ------ | ------------ | ------------------------------------------------ |
	| `_as`  | 区分重音     | `a` 和 `á` 被视为不同                            |
	| `_ci`  | 不区分大小写 | `a` 和 `A` 被视为相同                            |
	| `_cs`  | 区分大小写   | `a` 和 `A` 被视为不同                            |
	| `_ks`  | 区分假名     | 日语中平假名和片假名被视为不同                   |
	| `_bin` | 二进制比较   | 基于字符的二进制编码值进行比较，最快但最不“智能” |

##### (3)删除

删除数据库

语法：

`DROP DATABASE [IF EXISTS] 数据库名;`

- `IF EXISTS`   如果存在则删除

##### (4)使用

切换到要使用的数据库

语法：

`USE 数据库名;`

##### (5)修改

修改数据库的字符集和排序规则

> [!IMPORTANT]
>
> 修改后并不会影响数据库中已经存在的表的字符集和排序规则，只会改变新创建的表的字符集和排序规则

语法：

`ALTER DATABASE 数据库名 DEFAULT CHARACTER SET 新字符集 DEFAULT COLLATE 新排序规则;`

> [!WARNING]
>
> 由于在正式上线的系统中，随意修改数据库设置可能引发问题，所以最好一开始就设置正确，尽量不修改

#### 2.表操作

##### (1)查询

语法：

`SHOW TABLES;`

###### ②查询表结构

语法：

`SHOW CREATE TABLE 表名;`

创建表

语法：

```shell

	字段3 字段3类型(长度)[COMMENT '字段3注释'] [约束],
	......
	字段n 字段n类型(长度)[COMMENT '字段n注释'] [约束]

```

修改表名

- `IF EXISTS`   如果存在则删除

语法：

`TRUNCATE TABLE 表名;`

#### 3.字段操作

##### (1)修改

###### ①添加字段

语法：

`ALTER TABLE 表名 ADD 字段名 字段类型(长度)[COMMENT '注释'];`

###### ②修改字段类型

`ALTER TABLE 表名 MODIFY 字段名 新的字段类型(长度) [约束];`

###### ③修改字段名和字段类型

`ALTER TABLE 表名 CHANGE 旧字段名 新字段名 字段类型(长度)[COMMENT '注释'];`

###### ④给字段添加约束

语法：

`ALTER TABLE 表名 ADD CONSTRAINT 约束名 [约束]`

##### (2)删除

删除字段

语法：

`ALTER TABLE 表名 DROP 字段名;`

#### 5.约束

| 约束名称           | 作用                                     | 列级语法示例                                                 | 表级语法示例                                                 | 注意事项                                                     |
| :----------------- | :--------------------------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **NOT NULL**       | 字段值不能为空                           | `name VARCHAR(50) NOT NULL`                                  | 无表级形式，只能列级                                         | 插入时必须提供值，否则报错                                   |
| **UNIQUE**         | 字段值唯一（可多个NULL）                 | `phone VARCHAR(20) UNIQUE`                                   | `UNIQUE (phone)` 或 `UNIQUE (col1, col2)`（联合唯一）        | 允许NULL，NULL值不算重复；联合唯一表示组合值不能重复         |
| **PRIMARY KEY**    | 唯一标识一行，隐含NOT NULL + UNIQUE      | `id INT PRIMARY KEY`                                         | `PRIMARY KEY (id)` 或 `PRIMARY KEY (col1, col2)`（复合主键） | 一个表只能有一个主键，通常与AUTO_INCREMENT配合               |
| **FOREIGN KEY**    | 引用另一张表的列，保证参照完整性         | 不支持列级，必须表级定义                                     | `FOREIGN KEY (dept_id) REFERENCES departments(id)`           | 引用的列必须是主键或唯一键；会影响插入、更新、删除的约束检查 |
| **DEFAULT**        | 插入未指定值时自动填充默认值             | `status INT DEFAULT 1` `create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP` | 无表级形式，只能列级                                         | 默认值可以是常量或函数（如`CURRENT_DATE`），但函数在MySQL中需放在括号内 |
| **CHECK**          | 检查字段值是否满足条件                   | `age INT CHECK (age >= 18)`                                  | `CHECK (age >= 18)` 或 `CHECK (col1 > col2)`                 | **MySQL 8.0.16之前只解析不执行**；之后版本真正生效；其他数据库（PostgreSQL等）一直生效 |
| **AUTO_INCREMENT** | 自动为整数列生成递增序列（常与主键配合） | `id INT AUTO_INCREMENT PRIMARY KEY`                          | 无表级形式                                                   | 必须与主键或唯一键搭配；一个表只能有一个自增列；插入时可省略或给NULL；删除行后自增值不会回退 |
| **组合示例**       | 一个字段可带多个约束                     | `email VARCHAR(100) NOT NULL UNIQUE` `age INT NOT NULL DEFAULT 18` | 可在表级定义复合主键、复合唯一等                             | 约束顺序一般不影响，但自增必须紧跟在类型后                   |

#### 6.外键约束

##### (1)添加外键

语法：

 ` 外键名称 FOREIGN KEY (外键字段名) REFERENCES 主表(主表列名) [ON DELETE 约束] [ON UPDATE 约束]`

- `ON DELETE`   当父表记录被删除后执行的操作
- `ON UPDATE`   当父表记录被修改更新后执行的操作

外键删除/更新行为：

| 选项                 | 含义                            | 行为举例（以删除部门为例）                                   |
| :------------------- | :------------------------------ | :----------------------------------------------------------- |
| **RESTRICT**（默认） | 拒绝操作                        | 如果该部门下还有员工，就不允许删除这个部门。                 |
| **CASCADE**          | 级联操作                        | 删除部门时，同时删除该部门下的所有员工；更新部门ID时，自动更新员工表中的 `department_id`。 |
| **SET NULL**         | 设为空                          | 删除部门时，将原属于该部门的员工的 `department_id` 设为 `NULL`。 |
| **SET DEFAULT**      | 改为默认                        | 父表有变更时，子表将外键列设置乘一个默认的值                 |
| **NO ACTION**        | 与 RESTRICT 基本等价（MySQL中） | 同样拒绝操作。                                               |

> [!IMPORTANT]
>
> 内外键皆为**约束**中的操作

##### (2)删除外键

语法：

`ALTER TABLE 表名 DROP FOREIGN KEY 外键名称;`

##### (3)查询约束名

###### ①查看所有约束（主键、外键、唯一等）

```
SELECT * FROM information_schema.TABLE_CONSTRAINTS
WHERE TABLE_NAME = '表名' AND TABLE_SCHEMA = '数据库名';
```

###### ②查看外键的详细信息（引用哪个表、级联规则等）

```
SELECT * FROM information_schema.REFERENTIAL_CONSTRAINTS
WHERE CONSTRAINT_NAME = '约束名' AND TABLE_NAME = '表名';
```

###### ③查看哪些列参与了约束

```
SELECT * FROM information_schema.KEY_COLUMN_USAGE
WHERE TABLE_NAME = '表名' AND CONSTRAINT_NAME = '约束名';
```

###### ④查看表的索引（主键、唯一键、普通索引）

```
SHOW INDEX FROM 表名;
```

### 四.DML语句

#### 1.数据操作

##### (1)添加

###### ①给指定的字段添加数据

语法：

`INSERT INTO 表名(字段1,字段2...) VALUES (值1,值2...);`

###### ②给全部字段添加数据

语法：

`INSERT INTO 表名 VALUES (值1,值2...);`

###### ③批量添加数据

语法：

`INSERT INTO 表名(字段1,字段2...) VALUES(值1,值2...),(值1,值2...),(值1,值2...),...;`

`INSERT INTO 表名 VALUES(值1,值2...),(值1,值2...),(值1,值2...),...;`

> [!IMPORTANT]
>
> - 插入数据时，指定的字段需要与值的顺序是一一对应的，例如字段1对应值1，字段2对应值2等。
> - 字符串和日期型数据应当包含在单引号中。
> - 插入数据的大小应当在字段规定的范围内。

##### (2)更新

###### ①更新数据

语法：

`UPDATE 表名 SET 字段名1=值1,字段名2=值2，... [WHERE 条件];`

- `WHERE 条件`   条件例如`WHERE ID = 1`

> [!CAUTION]
>
> 如果没有条件则会修改整张表的所有数据
>
> 如果需要删除字段内某一个值，则可以将其替换为`NULL`

#### (3)删除

###### ①删除数据

语法：

`DELETE FROM 表名 [WHERE 条件];`

- `WHERE 条件`   条件例如`WHERE ID = 1`

### 五.SQL语句

#### 1.查询

##### (1)查询顺序：

```sql
SELECT(基本查询)
		字段列表
FROM
		表名列表
WHERE(条件查询)
		条件列表
GROUP BY(分组查询)
		分组字段列表
HAVING(分组查询)
		分组后条件列表
ORDER BY(排序查询)
		排序字段列表
LIMIT(分页ch)
		分页参数
```

##### (2)执行顺序

```sql
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
```

##### (1)基本查询

###### ①查询多个字段

语法：

`SELECT 字段1,字段2,字段3,... FROM 表名;`

`SELECT * FROM 表名;`

- `*`   代表查询所有字段

###### ②设置别名

语法：

`SELECT 字段1 [AS 别名1],字段2 [AS 别名2],... FROM 表名;`

> `AS` 可以省略
>
> 还可以使用`()`把数据括起来视为一体

###### ③去除重复记录

`SELECT DISTINCT 字段1,字段2,字段3,... FROM 表名;`

`SELECT DISTINCT * FROM 表名;`

- `*`   代表查询所有字段

##### (2)条件查询

语法：

`SELECT 字段1,字段2,字段3,... FROM 表名 [WHERE 条件];`

| 比较运算符             | 功能                                                  |
| ---------------------- | ----------------------------------------------------- |
| 字段名 >               | 大于                                                  |
| 字段名 >=              | 大于等于                                              |
| 字段名 <               | 小于                                                  |
| 字段名 <=              | 小于等于                                              |
| 字段名 =               | 等于                                                  |
| 字段名 <> 或 !=        | 不等于                                                |
| 字段名 BETWEEN … AND … | 在某个范围之内（包含边界）                            |
| 字段名 IN(…)           | 在in之后的列表中的值，多选一                          |
| 字段名 LIKE ‘占位符‘   | 模糊匹配(占位符\_匹配单个字符，占位符%匹配任意个字符) |
| 字段名 IS NULL         | NULL，空                                              |

| 逻辑运算符 | 功能                       |
| ---------- | -------------------------- |
| AND 或 &&  | 并且，多个条件同时成立     |
| OR 或 \|\| | 或者，多个条件任意一个成立 |
| NOT 或 !   | 非，不是                   |

##### (3)聚合函数

语法：

`SELECT 聚合函数(字段名) FROM 表名;`

| 聚合函数 | 功能     |
| -------- | -------- |
| count    | 统计数量 |
| max      | 最大值   |
| min      | 最小值   |
| avg      | 平均值   |
| sum      | 求和     |

> [!IMPORTANT]
>
> WHERE不能对聚合函数进行判断，而HAVING可以

##### (4)分组查询

语法：

`SELECT 字段名 FROM 表名 [WHERE 条件] GROUP BY 分组字段名 [HAVING 分组后过滤条件]`

> **SELECT 后的字段要么是分组字段（GROUP BY 后的字段），要么是聚合函数**，否则结果可能不符合预期（尤其在严格模式外）。

##### (5)排序查询

语法：

#### 2.外连接

##### (1)左外连接

语法：

#### 3.自连接

语法：

`SELECT 字段列表 FROM 表A 别名A JOIN 表A 别名B ON 条件...;`

#### 4.联合查询

将两次查询出来的结果直接合并

语法：

`SELECT 字段列表 FROM 表A [WHERE ...] UNION [ALL] SELECT 字段列表 FROM 表B [WHERE ...]`

- `ALL`   加上会导致数据在没有去重的情况下显示

> [!CAUTION]
>
> 联合查询的多张表的列数必须保持一致，字段类型也需要保持一致

#### 5.子查询

语法：

`SELECT * FROM t1 WHERE column1=(SELECT column1 FROM t2);`

- `SELECT * FROM t1 WHERE column1=`   外部语句，也可以是INSERT/UPDATE/DELETE/SELECT中的任何一个

##### (1)标量子查询

子查询返回的只有一行一列的数据

常用的操作符：`= <> > >= < <=`

语法：

`SELECT 父查询字段名 FROM 父表名 WHERE 父查询字段名=(SELECT 子查询字段名 FROM 子表名 WHERE ...)`

##### (2)列子查询

子查询返回的结果是一列（可以是多行）

常用的操作符：`IN、NOT IN、ANY、SOME、ALL`

- `ANY`   子查询返回列表中，有任意一个满足即可
- `SOME`   与ANY等同，使用SOME的地方都可以使用ANY
- `ALL`   子查询返回列表的所有值都必须满足

示例：

`SELECT * FROM emp WHERE salary > ALL (子表查询结果)`

##### (3)行子查询

子查询返回的结果是一行（可以是多列）

常用的操作符：`=、<>、IN、NOT IN`

示例：

`SELECT * FROM emp WHERE (salary,managerid) = (SELECT salary,managerid FROM emp WHERE ...)`

##### (4)表子查询

子查询返回的结果是多行多列

常用的操作符：`IN、EXISTS`

- `EXISTS`   获取到一个数据就停止

示例：

`SELECT * FROM emp WHERE (job,salary) IN (子查询结果)`

> [!IMPORTANT]
>
> 可以把子查询的数据作为临时表放入`表名`位置，使用`()`括起来

### 十.事务

#### 1.事务操作

##### (1)查看/设置事务提交方式

###### ①查看事务提交方式

语法：

`SELECT @@autocommit;`

> 查看当前事务提交方式，1为自动，0为手动

###### ②显式事务

语法：

`SET @@autocommit=0;`

> 将提交方式改为手动提交

###### ③隐式事务

语法：

`BEGIN`

> `START TRANSACTION`是MySQL的标准语法

> [!CAUTION]
>
> - `BEGIN` (显式事务)：
> 	- 事务边界非常清晰：从 `BEGIN` 开始，到 `COMMIT` 或 `ROLLBACK` 结束。
> 	- 即使你忘了写 `COMMIT`，断开连接时系统可能会回滚（取决于严格的配置），但通常它要求你必须显式结束。
> - `SET autocommit=0` (隐式事务)：
> 	- 从执行 `SET autocommit=0` 的那一刻起，**下一条语句**就开始一个新事务。
> 	- 除非你执行 `COMMIT`，否则事务会一直保持打开状态。
> 	- 这容易导致“长事务”（忘记提交，锁一直不释放），是生产环境中比较常见的隐患。

##### (2)提交事务

语法：

`COMMIT;`

##### (3)回滚事务

语法：

`ROLLBACK;`

- 原子性：事务时不可分割的最小操作单元，要么全部成功，要么全部失败
- 一致性：事务完成时，必须使所有的数据都保持一致状态
- 隔离性：数据库系统提供的隔离机制，保证事务在不受外部并发操作影响的独立环境下运行
- 持久性：事务一旦提交或回滚，他对数据库中的数据的改变就是永久的

#### 3.并发事务问题

| 问题       | 描述                                                         |
| ---------- | ------------------------------------------------------------ |
| 脏读       | 一个事务读到另一个事务还没有提交的数据                       |
| 不可重复读 | 一个事务先后读取同一条记录，但两次读取的数据不同，称之为不可重复读，主要存在于事务A和事务B同时对某个表中的数据进行操作时，事务A查询过一次后准备再次查询时事务B刚好又对这个表中的数进行更新，此时事务A查询到的数据就不同了 |
| 幻读       | 在一个事务内，两次执行相同条件的查询，返回的结果集行数不一致 |

#### 4.事务隔离

##### (1)级别

| 隔离级别         | 脏读   | 不可重复读 | 幻读                   | 说明                                                         |
| :--------------- | :----- | :--------- | :--------------------- | :----------------------------------------------------------- |
| READ UNCOMMITTED | 可能   | 可能       | 可能                   | 最低级别，一个事务可读到另一个未提交事务的修改（脏读）       |
| READ COMMITTED   | 不可能 | 可能       | 可能                   | 大多数数据库默认级别，只能读到已提交的数据，但同一事务内两次查询可能结果不同 |
| REPEATABLE READ  | 不可能 | 不可能     | 可能（部分数据库解决） | MySQL InnoDB 默认级别，通过 MVCC 保证同一事务内多次读取一致，但可能产生幻读 |
| SERIALIZABLE     | 不可能 | 不可能     | 不可能                 | 最高级别，强制事务串行执行，性能最低                         |

##### (2)查看事务隔离级别

语法：

`SELECT @@TRANSACTION_ISOLATION;`

##### (3)设置事务隔离级别

语法：

`SELECT [SESSION | GLOBAL] TRANSACTION ISOLATION LEVEL {READ UNCOMMITTED| READ COMMITTED | REPEATABLE READ | SERIALZABLE}`

- `SESSION`   仅对于当前客户端窗口有效
- `GLOBAL`   针对于所有客户端窗口有效

### 十一.Python操作数据库

语法：

```sql
 #导入库
 import pymysql
 
 #连接mysql数据库
 conn = pymysql.connect(
    host='8.141.85.194',   #主机名
    user='pai',            #用户名
    password='',             #密码
    port=3306,             #端口
    database='mydb',       #要操作的数据库
    charset='utf8mb4'      #选择字符编码方式
)

#创建游标
cursor=conn.cursor()       

#需要执行的SQL语句
cursor.execute('')         

#获取返回值（如果只有少量数据）
result = cursor.fetchall() 

#获取返回值（如果有大量数据）
result = cursor.fetchone()

conn.commit()

conn.rollback()

#关闭游标
cursor.close()

#关闭连接
conn.close()
```
