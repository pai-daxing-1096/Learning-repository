

# Linux系统学习笔记

## 一.Linux简介

### 1.Linux系统架构：

**Linux操作系统=Linux内核+一套系统调用的API接口**

硬件→Linux内核→系统调用接口→shell+函数库→应用程序

1. 操作系统内核直接参考硬件规格写成，所以同一个操作系统程序不能安装在不一样的硬件架构上，主流的架构有 x86 架构和主要用于手机与平板设备的 ARM 架构
2. 操作系统只是管理整个硬件资源，包括 CPU 、内存、输入输出设备及文件系统等。如果没有其他应用程序辅助，操作系统只能是让电脑主机 Ready 而已，并没有其他功能。
3. 应用程序的开发都是参照操作系统提供的 API ，所以该应用程序只能在该操作系统上面运行，不能在其他操作系统上运行，这就是为什么大家在下载应用软件的时候会让你选择是什么平台的安装

### 2.文件系统层次结构标准

| 目录   | 作用                                                         |
| ------ | ------------------------------------------------------------ |
| /bin   | 存放**基本用户命令**，如`ls`、`cp`、`cat`等，这些命令在单用户模式或系统启动早期都必须可用，所有用户都可以执行 |
| /boot  | 存储**启动系统所需的文件**，包括Linux内核、初始内存盘、引导加载文程序配置文件 |
| /dev   | 包含**设备文件**，代表设备中的**硬件设备**，如硬盘(`/dev/sda`)、终端(`\dev\tty`)、随机数生成器(`/dev/random`)。应用程序通过读写这些文件来访问硬件 |
| /etc   | 存放**系统配置文件**，几乎所有的**全局配置**都位于此，如网络设置(`/etc/network`)、用户账户信息(`/etc/passwd`)、软件包配置(`/etc/yum.conf`) |
| /home  | **普通用户的个人主目录**所在地，每个用户都有一个子目录(如`/home/pai`)，存放用户的文档、下载、配置(隐藏文件如`.ssh`)等 |
| /lib   | **存储共享库**和**内核模块**，供`/bin`和`/sbin`中的程序调用，也包含某些驱动程序 |
| /media | 作为**可移动介质的挂载点**，如CD/DVD、USB闪存盘、外界硬盘，系统自动挂载时通常在此创建子目录 |
| /mnt   | 用于**临时挂载**其它文件系统(如网络存储、额外分区)，通常由管理员手动挂载，作为临时挂载点 |
| /opt   | 存放可选的**第三方应用程序软件包**，软件通常会以完整目录形式安装于此，避免与系统默认程序冲突 |
| /proc  | **虚拟文件系统**，提供内核与进程的实时信息（存在于内存中）。例如 `/proc/cpuinfo` 查看 CPU 信息，`/proc/<pid>/` 查看进程详情。也可通过写入某些文件（如 `/proc/sys/`）动态修改内核参数。 |
| /root  | **超级管理员的个人主目录**，正常情况下普通用户无权访问，用于储存root用户的个性化配置和重要脚本 |
| /run   | **临时文件系统**，用于储存系统启动以来的运行时数据，如进程PID文件、当前登录会话、服务状态等 |
| /sbin  | 存放**系统管理命令**，通常仅供root用户使用，如`flask`、`mount`、`systemctl`，这些通常命令用于系统维护、启动修复等 |
| /srv   | 存放**本系统提供的服务数据**，即服务（如 HTTP、FTP、RSync）所管理的文件。例如网页服务器可将网站文件放在 `/srv/www/` 下 |
| /sys   | 虚拟文件系统（`sysfs`），**暴露内核对象、设备及驱动信息**，比 `/proc` 更加结构化。用于设备管理和电源管理（如 `/sys/class/`） |
| /tmp   | 存放**临时文件**，任何用户和程序都可以读写此目录，但文件通常不保证持久性 |
| /usr   | **用户系统资源**和**应用程序**的主要存放地，其下的子目录类似根目录（如 `/usr/bin`、`/usr/lib`、`usr/share`），但存放的是非启动必需的程序、库、文档等。大部分软件默认安装于此 |
| /var   | 存放**经常变化的数据**，例如日志文件（`/var/log`）、邮件队列（`/var/mail`）、打印队列（`/var/spool`）、临时缓存（`/var/cache`）、系统运行时的状态文件（`/var/lib`）。 |

### 3.相对路径、绝对路径和特殊路径符

#### (1)相对路径和绝对路径

- 绝对路径：以根目录为起点，描述路径的一种写法，路径描述以`/`为开头

	例：`cd /home/itheima/Desktop`

- 相对路径：以当前目录为起点，描述路径的一种写法，路径描述无需以`/`为开头

​	例：`cd Desktop`

#### (2)特殊路径符

- `.`    表示当前目录，比如`cd ./Desktop`表示切换到当前目录下的Desktop目录内，和`cd Desktop`效果一致，其可以与`/`连同使用[^4]
- `..`   表示上一级目录，比如：`cd .. `即可切换到上一级目录，`cd ../..`切换到**上二级目录**，其可以与路径`/`直接连接使用[^4]
- `~`    表示HOME目录，比如：`cd ~`即可切换到HOME目录或`cd ~/Desktop`，切换到HOME内的Desktop目录

## 三.Linux命令入门

- 命令行：即Linux终端（Terminal)，是一种**命令提示符页面**。以纯**字符**的形式操作系统，可以使用各种字符化命
	令对系统发出操作指令。
- 命令：即Linux程序。一个命令就是一个**Linux的程序**。**命令没有图形化页面**，可以在命令行（终端中）提供字符化的反馈

### 1.Linux命令基础格式

无论是什么命令，用于什么用途，在Linux中，命令有其通用的格式：

  `command [OPTION] [parameter]`

- **command:命令本身**
- **options：[可选，非必填]命令的一些选项，可以通过选项控制命令的行为细节**

> 语法中的`[]`表示可选的意思

示例：

  `ls -l /home/itheima`：

- 其中`ls`是命令本身，`-l`是选项,`/home/itheima`是参数

	- **意思是以列表的形式，显示`/home/itheima`目录内的内容**

	`cp -r test1 test2`：

- 其中`cp`是命令本身，`-r`是选项，`test1`和`test2`是参数

	- **意思是复制文件夹`test1` 成为`test2`**

## 四.Linux命令

### 1.文件及目录的操作命令

#### (1)ls命令

ls命令的作用是列出目录下的内容

语法：

`ls [OPTION] [FILE]`

- `[OPTION]`   参数
- `[FILE]`   文件/路径

| OPTION | 作用                                                 | 示例              |
| ------ | ---------------------------------------------------- | ----------------- |
| `-a`   | 表示all的意思，即列出全部文件(包括隐藏的文件/文件夹) | `ls -a /home/pai` |
| `-l`   | 表示以列表的形式展示内容，并展示更多信息             | `ls -l /home/pai` |
| `-h`   | 表示以易于阅读的形式，列出文件大小，如K、M、G        | `ls -h /home/pai` |
| `-i`   | 用于检查文件inode                                    | `ls -i /home/pai` |
| `-t`   | 按照文件修改的时间来排序                             | `ls -t /home/pai` |

> 命令可以连续使用，例如：`ls -la /home/pai`

#### (2)cd命令

cd命令可以更改当前的工作目录

语法:

`cd [dir]`

- `dir`   目标目录

| dir  | 作用                 | 示例    |
| ---- | -------------------- | ------- |
| `~`  | 回到用户的Home目录   | `cd ~`  |
| `.`  | 代指当前目录         | `cd .`  |
| `..` | 回到上一级位置       | `cd ..` |
| `-`  | 回到上一次所在的目录 | `cd -`  |

> 若未使用dir则默认回到当前用户的家目录

#### (3)pwd命令

pwd命令可以查看当前所在的工作目录

语法:`pwd`

#### (4)tree命令

展示当前目录的树状图

语法：

`tree [OPTHON]... [FILE]`

- `[OPTION]`   参数（可连续使用多个参数）
- `[FILE]`   文件/路径

| OPTION     | 作用                                 | 示例                  |
| ---------- | ------------------------------------ | --------------------- |
| `-L level` | 限制目录遍历最大深度（level为层数）  | `tree -L 2 /home/pai` |
| `-a`       | 显示所有文件，包括隐藏文件           | `tree -a /home/pai`   |
| `-d`       | 只显示目录                           | `tree -d /home/pai`   |
| `-f`       | 在每个文件或目录前显示完整的相对路径 | `tree -f /home/pai`   |
| `-h`       | 以人类易读的方式显示文件大小         | `tree -h /home/pai`   |
| `--du`     | 显示每个子目录的磁盘使用情况         | `tree --du /home/pai` |
| `-p`       | 打印每个文件的权限信息               | `tree -p /home/pai`   |

> 在CentOS等系统中需要下载该命令：
>
> `yum install tree-y`

#### (5)mkdir命令

通过mkdir命令可以创建新的目录（文件夹）

语法：

`mkdir [OPTION]... DIRECTORY...`

- `[OPTION]...`   参数（可连续使用多个参数）

- `DIRECTORY...`   创建的目录的名称或路径（可连续指定多个名称或路径）

| OPTION | 作用                                                     | 示例                       |
| ------ | -------------------------------------------------------- | -------------------------- |
| `-p`   | 递归，自动创建不存在的父目录，适用于创建连续多层级的目录 | `mkdir -p /home/pai/hello` |
| `-v`   | 可以直观地看见创建了哪些目录                             | `mkdir -v /home/pai/hello` |

#### (6)touch命令

可以通过touch命令创建文件或修改时间戳

语法：

`touch [OPTION]... FILE...`

- `[OPTION]...`   参数（可连续使用多个参数）
- `FILE...`   创建的文件的名称（可连续指定多个）

| OPTION | 作用                                   | 示例                                 | 结果                                                         |
| ------ | -------------------------------------- | ------------------------------------ | ------------------------------------------------------------ |
| `-c`   | 不创建任何文件                         | `touch -c hello.txt`                 | 如果hello.txt文件不存在，则静默跳过，不创建新文件，如果hello.txt文件存在，则正常更新时间戳 |
| `-a`   | 刷新最近访问时间和最近改动时间         | `touch -a hello.txt`                 | 刷新hello.txt文件的最近访问时间和最近改动时间                |
| `-m`   | 刷新最近访问、最近更改和最近改动的时间 | `touch -m hello.txt`                 | 刷新hello.txt文件的最近更改和最近改动的时间                  |
| `-t`   | 修改时间戳至指定时间                   | `touch -t 202604132213.12 hello.txt` | 将hello.txt文件的最近访问时间和最近改动时间改为2026-04-13-22:13:12 `[[CC]YY]MMDDhhmm[.ss]` |

#### (7)cat命令

可以通过cat命令查看内容

语法：

`cat [OPTION]... FILE...`

- `[OPTION]...`   参数（可连续使用多个参数）
- `FILE...`   需要查看的文件的名称（可连续指定多个）

| OPTION | 作用         | 示例               | 结果                                            |
| ------ | ------------ | ------------------ | ----------------------------------------------- |
| `-n`   | 显示行号     | `cat -n hello.txt` | 为每一行前添加行号                              |
| `-A`   | 显示所有符号 | `cat -A hello.txt` | 显示出文件内的所有符号（$、^I等会被隐藏的符号） |

#### (8)less命令

同样可以查看文件内容，如果文件过多，将会一页一页地展示，除此之外，还有向上翻页的功能

语法：

`less FILE...`

- `FILE...`   需要查看的文件的名称（可连续指定多个）

| 操作介绍            | 作用         |
| ------------------- | ------------ |
| `[space]`           | 向下翻一页   |
| `b`                 | 向上翻一页   |
| `Enter`             | 翻一行       |
| `q`                 | 退出         |

#### (9)file命令

输出文件的类型，例如ASCII text，表示这是一个ASCII编码的文本文件。

语法：

`file [OPTION]... FILE...`

- `[OPTION]...`   参数（可连续使用多个参数）
- `FILE...`   需要查看的文件的名称（可连续指定多个）

| OPTION | 作用                                     | 示例                | 结果                                                         |
| ------ | ---------------------------------------- | ------------------- | ------------------------------------------------------------ |
| `-b`   | 仅显示文件类型，不显示文件名             | `file -b hello.txt` | 只显示文件类型，没有`hello.txt:`前缀                         |

#### (11)cp命令

用于复制文件\文件夹

语法：
 `cp [OPTION]... SOURCE... DIRECTORY`

- `[OPTION]...`   参数（可连续使用多个参数）
- `SOURCE...`   源文件路径（可连续指定多个文件）
- `DIRECTORY`   目标目录（必须已经存在，否则会报错）

| OPTION      | 作用                                                         | 示例                           | 结果                                                         |
| ----------- | ------------------------------------------------------------ | ------------------------------ | ------------------------------------------------------------ |
| `-i`        | 覆盖前先询问用户（y确认）                                    | `cp -i a.txt b.txt`            | a.txt的内容被复制到b.txt中，且复制前会先进行确认             |
| `-f`        | 不询问，直接覆盖（会覆盖 -i 的效果）                         | `cp -f a.txt b.txt`            | a.txt的内容被复制到b.txt中，且复制前不会进行确认             |
| `-n`        | 如果目标文件已存在，则不进行复制                             | `cp -n a.txt b.txt`            | 因为b.txt文件存在，所有a.txt的内容没有被复制到b.txt中        |
| `-u`        | 只有当源文件比目标文件新，或目标不存在时才复制               | `cp -u a.txt b.txt`            | 因为a.txt文件修改时间比b.txt文件更晚（更新），所以a.txt文件的内容被复制到了b.txt文件内 |
| `-v`        | 打印每个复制操作的信息                                       | `cp -v a.txt b.txt`            | 显示过程                                                     |
| `-r` / `-R` | 复制目录及其内部所有内容（两者基本等价）                     | `cp -r ~/hello/ ~/world/`      | 将 hello 目录复制到 world 目录下（即 ~/world/hello/）        |
| `-T`        | 强制把目标（不是源文件，是目标位置）当作普通文件处理。如果目标是一个已存在的目录，则报错（防止意外将文件复制到目录内部） | `cp -T a.txt /home/pai/hello/` | 报错，因为hello/是一个目录                                   |
| `-t`        | 多文件复制                                                   | `cp -t ~/hello/ a.txt b.txt`   | 将a.txt文件和b.txt文件都复制到~/hello/下                     |
| `-p`        | 保留原始文件的修改时间、访问时间、权限、所有者等             | `cp -p a.txt ~/hello/`         | 将a.txt文件复制到~/hello/下，但是修改时间、访问时间、权限、所有者等没有变化 |

| 类型\情况 | 存在                         | 不存在                 |
| --------- | ---------------------------- | ---------------------- |
| 文件      | 将源文件内容复制到指定文件中 | 创建指定文件后复制进去 |
| 目录      | 将源文件复制到指定目录中     | 报错                   |

> [!WARNING]
>
> 复制目录时**必须加 `-r` 或 `-R`**

#### (12)mv命令

用于移动文件\文件夹

语法：`mv [OPTION]... SOURCE... DIRECTORY`

- `[OPTION]...`   参数（可连续使用多个参数）
- `SOURCE...`   源文件路径（可连续指定多个文件）
- `DIRECTORY`   目标目录（必须已经存在，否则会报错）

| OPTION | 作用                                                         | 示例                          | 结果                                                         |
| ------ | ------------------------------------------------------------ | ----------------------------- | ------------------------------------------------------------ |
| `-i`   | 在覆盖目标文件前提示用户确认（y确认）                        | `mv -i a.txt c.txt`           | 因为c.txt文件不存在，所以将a.txt文件改名为c.txt              |
| `-f`   | 不询问，直接覆盖目标文件（会覆盖 -i 的效果）                 | `mv -f a.txt b.txt`           | 将a.txt文件的名称重命名为b.txt，同时a.txt文件消失            |
| `-n`   | 如果目标文件已存在，则不会进行移动操作                       | `mv -n a.txt b.txt`           | 因为b.txt文件存在，所以没有进行任何操作                      |
| `-u`   | 只有当源文件比目标文件更新（修改时间），或目标文件不存在时，才进行移动 | `mv -u a.txt b.txt`           | 因为a.txt文件修改时间比b.txt文件更晚（更新），所以a.txt文被重命名为b.txt，同时a.txt文件消失 |
| `-v`   | 打印每个移动操作的详细信息                                   | `mv -v a.txt b.txt`           | 显示过程                                                     |
| `-b`   | 如果目标文件已存在，会在覆盖前自动为其创建一个备份（备份文件默认添加 `~` 后缀） | `mv -b a.txt b.txt`           | 因为b.txt文件存在，所以会先将b.txt文件重命名为b.txt~，再将 a.txt文件重命名为b.txt，a.txt文件消失 |
| `-t`   | 可将目标目录写在前面，方便多文件移动（源文件可放在后面）     | `mv -t ~/hello/ a.txt b.txt ` | 将a.txt文件和b.txt文件都移动到~/hello/目录下                 |
| `-T`   | 将目标当作普通文件，而非目录。如果目标是一个已存在的目录，则报错 | `mv -T a.txt ~/hello/`        | 报错                                                         |
| `-S`   | 与 -b或连用，指定备份文件的后缀（默认是~）                   | `mv -b -S .bak a.txt b.txt`   | 将a.txt文件的内容覆盖至到b.txt文件中，如果b.txt文件存在，会先将b.txt文件备份为b.txt.bak文件再移动 |

| 类型\情况 | 存在                                             | 不存在                   |
| --------- | ------------------------------------------------ | ------------------------ |
| 文件      | 覆盖目标文件内容但不改变目标文件名称，源问价消失 | 先创建指定文件再覆盖内容 |
| 目录      | 将文件移动至目标目录内                           | 报错                     |

> [!CAUTION]
>
> `-i`、`-f`、`-n` 三个选项互斥，如果同时指定，只有最后一个选项生效
>
> 跨文件系统移动文件时，`mv` 实际上会执行“先复制后删除”的操作，这会导致文件的索引节点（inode）编号发生改变，速度也会比同分区内移动慢不少

#### (13)rm命令

用于删除文件\文件夹

语法：`rm [OPTION]... [FILE]...`

- `[OPTION]...`   参数（可连续使用多个参数）
- `[FILE]...`   需要删除的文件的名称（可连续指定多个）

| OPTION     | 作用                                                   | 示例             | 结果                          |
| ---------- | ------------------------------------------------------ | ---------------- | ----------------------------- |
| `-i`       | 删除前逐一询问确认（y确认）                            | `rm -i a.txt`    | 删除a.txt文件，删除前提示     |
| `-f`       | 忽略不存在的文件，不提示确认（会覆盖 -i）              | `rm -f a.txt`    | 删除a.txt文件，删除前没有提示 |
| `-r`或`-R` | 删除目录及其内部所有内容（**删除目录时必须加此参数**） | `rm -r ~/hello/` | 删除hello/目录                |
| `-v`       | 打印每个删除操作的过程                                 | `rm -v a.txt`    | 显示过程                      |

> [!CAUTION]
>
> `rm` 默认**不删除目录**，必须加 `-r` 才能删除目录。
>
> `-f` 会抑制错误信息（比如文件不存在时不报错）且不询问确认。

### 2.查找命令

#### (1)which命令

语法：

`which command`

#### (2)find命令

用于搜索指定文件

语法：

`find 起始路径 -name “被查找文件名” -size +/-n[kMG]`

- `+、-`   表示大于小于
- `n`   表示大小数字
- `kMG`   表示大小单位，`k`表示`kb`,`M`表示`Mb`，`G`表示`Gb`
